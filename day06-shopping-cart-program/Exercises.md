# 📝 Day 06 Exercises - Shopping Cart Program

## 📖 Overview

These exercises reinforce the concepts learned while building the Shopping Cart Program.

Topics covered:

- Scanner
- User Input
- Variables
- String
- int
- double
- char
- Arithmetic Operations
- Console Output

---

# Exercise 1 — Product Information

Ask the user to enter:

- Product Name
- Price
- Quantity

Display all the entered information.

---

# Exercise 2 — Calculate Total

Using the user's input, calculate:

```
Total = Price × Quantity
```

Display the result.

---

# Exercise 3 — Receipt Generator

Display the following format:

```
========== RECEIPT ==========

Item      : Pizza
Price     : $12.99
Quantity  : 3

Total     : $38.97

=============================
```

---

# Exercise 4 — Multiple Products (Manual)

Create variables for:

- Burger
- Fries
- Drink

Calculate the grand total.

---

# Exercise 5 — Currency Practice

Use different currency symbols.

Example:

```
$
€
₹
£
¥
```

Display the total using each currency.

---

# Exercise 6 — Discount

Ask for:

- Price
- Quantity

Calculate the total.

Then apply a 10% discount.

Display:

- Original Total
- Discount
- Final Total

---

# Exercise 7 — Tax Calculator

Assume GST = 18%

Calculate:

```
Tax Amount

Final Price
```

---

# Exercise 8 — Grocery Bill

Ask the user to buy:

- Apples
- Milk
- Bread

Display a complete receipt.

---

# Challenge ⭐

Create a Shopping Cart Program that calculates:

- Subtotal
- Discount
- Tax
- Final Bill

---

# Revision Questions

1. Which Scanner method reads text?

2. Which Scanner method reads integers?

3. Which Scanner method reads decimal numbers?

4. Why is the price stored as a double?

5. Why is quantity stored as an int?

6. Why is the currency stored as a char?

7. What formula calculates the total?

8. Why should a Scanner be closed?

9. Which operator multiplies two numbers?

10. Which data type stores the product name?

---

# Expected Learning Outcome

After completing these exercises, you should be comfortable creating simple billing programs, accepting multiple user inputs, performing calculations, and displaying formatted receipts.
